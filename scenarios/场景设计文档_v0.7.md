## 场景六：反面教材

### 场景描述
展示好与坏的实践对比。

### ❌ 坏案例：不用Skill

```
问题1：不用Skill，直接写代码
┌─────────────────────────────────────────────────────────────────────┐
│ 王五不用Skill，直接用Claude生成代码                                  │
│ 结果：代码质量参差不齐                                              │
│ 问题：不符合验收标准，被发起人拒绝                                    │
│ 惩罚：信誉分-1，需要重做                                            │
└─────────────────────────────────────────────────────────────────────┘

问题2：用别人的Skill但不检查
┌─────────────────────────────────────────────────────────────────────┐
│ 王五导入李四的Skill，但直接提交不做检查                              │
│ 结果：代码有Bug，被发起人拒绝                                        │
│ 问题：没有按照Skill的质量标准自检                                    │
│ 教训：Skill是指导，不是代替思考                                    │
└─────────────────────────────────────────────────────────────────────┘
```

### ✅ 好案例：用Skill

```
好实践1：用Skill并按流程执行
┌─────────────────────────────────────────────────────────────────────┐

│ 步骤1：分析需求 → 步骤2：设计模型 → 步骤3：写代码 → 步骤4：写测试 │
│ 结果：代码质量高，一次验收通过                                      │
│ 收益：¥1,050 到账，信誉分+1                                       │
└─────────────────────────────────────────────────────────────────────┘

好实践2：优化别人的Skill
┌─────────────────────────────────────────────────────────────────────┐
│ 王五使用李四的Skill，发现可以优化的地方                            │
│ 改进：添加了缓存优化和性能提升                                     │
│ 成果：创建了自己的优化版Skill，分享到社区                          │
│ 收益：获得Skill使用者的好评                                        │
└─────────────────────────────────────────────────────────────────────┘
```
