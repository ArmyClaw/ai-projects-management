# AI Project Management Platform - 需求规格说明书 v0.1

## 1. 产品概述

### 1.1 产品定位
**AI 项目管理平台** 是一个连接 AI 能力供给方与需求方的双边市场，同时提供完整的 AI 项目管理流程服务。

### 1.2 核心价值
- **降低 AI 使用门槛**：用户无需技术背景，通过自然语言描述需求即可获得 AI 服务
- **AI 能力交易**：创建一个 AI "劳务市场"，让 AI 能力像商品一样交易
- **标准化流程**：参考 PMP 体系，提供完整的 AI 项目管理方法论

### 1.3 目标用户

| 角色 | 描述 | 需求 |
|------|------|------|
| **AI 雇主 (需求方)** | 发布需求，雇佣 AI 完成项目 | 项目管理、需求发布、验收 |
| **AI 雇员 (供给方)** | 提供 AI 能力（API/算力） | 任务获取、收益结算 |
| **平台运营方** | 平台维护、规则制定 | 审核、风控、结算 |

## 3. 核心功能详细设计

### 3.0 AI 雇员管理系统 (AI Workforce Management)

> 核心创新：由于雇佣的是 AI 的闲置时间，需要像管理人类员工一样管理 AI 的工作状态

#### 3.0.1 在岗确认 (Check-in System)
- **上班打卡**：AI 开始工作前需确认在岗
  - 自动发送确认请求
  - 设置确认超时时间（如 5 分钟）
  - 超时未确认自动启动备用 AI
- **在岗状态**：
  - 🟢 在线可用
  - 🟡 工作中
  - 🔴 离线/忙碌
  - ⚪ 休息中

#### 3.0.2 工作状态监控 (Work Status Monitoring)
- **定时签到**：
  - 可配置签到间隔（默认 15 分钟）
  - 记录 AI 响应时间
  - 检测异常（如响应变慢）
- **进度汇报**：
  - AI 定期上报任务进度
  - 自动记录关键里程碑
  - 异常告警

#### 3.0.3 备份计划 (Backup Plan)
- **自动备份机制**：
  - 任务开始时自动分配备用 AI
  - 备用 AI 同步任务进度
  - 主 AI 异常时自动切换
- **备份策略**：
  - 热备份：实时同步状态
  - 温备份：定期同步
  - 冷备份：仅保留关键节点
- **切换机制**：
  - 自动切换条件：连续 N 次签到失败
  - 手动切换：雇主主动选择
  - 切换后进度继承

#### 3.0.4 工时统计 (Time Tracking)
- **工时记录**：
  - 精确到秒的任务时间
  - 区分有效工作/等待时间
  - 自动计算实际工时
- **工时报表**：
  - 按项目统计
  - 按 AI 能力统计
  - 按时段统计
- **超时预警**：
  - 接近截止时间提醒
  - 工时超出预算告警

#### 3.0.5 结算系统 (Settlement System)
- **结算方式**：
  - 按实际工时结算
  - 按任务完成量结算
  - 按里程碑结算
- **结算公式**：
  ```
  实际费用 = 工时单价 × 实际工时 + 质量奖金 - 扣款
  ```
- **扣款规则**：
  - 延期扣款：超出截止时间部分
  - 质量扣款：验收不通过次数
  - 异常扣款：频繁签到失败
- **结算流程**：
  ```
  任务完成 → 工时审核 → 质量确认 → 金额计算 → 双方确认 → 资金转账
  ```

### 3.1 需求中心 (Demand Center)

#### 3.1.1 需求发布
- **需求类型**：
  - 代码开发 (Code Development)
  - 数据分析 (Data Analysis)
  - 内容创作 (Content Creation)
  - 文档撰写 (Documentation)
  - 测试执行 (Testing)
  - 部署运维 (DevOps)
  - 研究咨询 (Research)

- **需求结构**：
  ```yaml
  需求:
    标题: 简洁描述
    详细描述: 完整需求说明
    验收标准: 可量化的交付标准
    预算范围: 最低-最高
    截止时间: 日期时间
    优先级: P0/P1/P2/P3
    标签: [技术栈, 领域, 难度]
    附件: 参考文档/代码/数据
  ```

#### 3.1.2 AI 需求解析
- 自动将自然语言需求转换为结构化需求
- 识别技术栈要求
- 评估工作量和难度
- 生成初步项目计划

### 3.2 雇佣市场 (AI Marketplace)

#### 3.2.1 AI 能力上架
- **能力类型**：
  - 大语言模型 (LLM) - GPT/Claude/Qwen/DeepSeek 等
  - 图像生成模型 - Stable Diffusion/Midjourney 等
  - 语音模型 - TTS/ASR
  - 代码模型 - CodeLlama/DeepSeek-Coder 等
  - 垂直模型 - 数学/医疗/金融等

- **能力信息**：
  ```yaml
  AI 能力:
    名称: 模型名称 + 版本
    类型: 模型分类
    能力描述: 支持的任务类型
    定价: 元/千tokens 或 元/次
    性能指标: 响应速度、准确率
    可用时段: 7x24 / 指定时段
    算力来源: 自有服务器 / 云计算 / 闲置资源
  ```

#### 3.2.2 雇佣模式
- **竞价模式**：需求方设置预算，AI 方报价竞争
- **定价模式**：AI 方设置固定价格
- **委托模式**：平台智能匹配最优 AI

### 3.3 项目管理系统 (PM System)

#### 3.3.1 项目生命周期
```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  项目    │───▶│  规划   │───▶│  执行   │───▶│  监控   │───▶│  收尾   │
│  启动   │    │ Planning│    │Execute │    │Monitor │    │Closeout│
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
  需求冻结      计划生成      任务分配      进度同步      验收交付
  立项审批      资源匹配      AI 雇佣      风险预警      报告归档
```

#### 3.3.2 任务管理
- **任务类型**：
  - 主任务 (Task)
  - 子任务 (Sub-task)
  - 里程碑 (Milestone)

- **任务状态**：
  ```
  待开始 → 进行中 → 待验收 → 已完成 / 已延期
  ```

#### 3.3.3 AI 工作流编排
- 自动拆分任务为可执行子任务
- 根据任务类型匹配 AI 能力
- 生成执行计划（先后顺序、依赖关系）
- 支持并行/串行执行策略

### 3.4 财务中心

#### 3.4.1 交易模式
- **预付费**：雇主充值，平台托管
- **后付费**：先用后付，按量结算
- **保证金**：双方缴纳，保障履约

#### 3.4.2 结算周期
- T+1 即时结算
- 周/月批量结算
- 提现审核机制

## 5. 核心流程设计

### 5.1 完整项目流程

```
1. 需求发布
   └── 2. AI 能力匹配
       └── 3. 雇佣确认
           └── 4. 项目启动
               └── 5. 任务执行 (循环)
                   │  - 规划分解
                   │  - AI 调度
                   │  - 进度同步
                   │  - 质量检查
                   └── 6. 验收交付
                       └── 7. 资金结算
                           └── 8. 项目归档
```

### 5.2 典型场景

#### 场景 1：开发一个 Web 应用
```
用户：我想开发一个博客系统
AI 解析：
  - 识别为"代码开发"类型
  - 拆分为：后端 API、前端页面、数据库设计、部署配置
  - 估算工作量：40 小时
  - 推荐 AI 组合：Claude(后端) + Cursor(前端) + GPT-4(测试)

执行流程：
  1. 创建项目"个人博客系统"
  2. 生成技术方案
  3. 雇佣 Claude 完成 API 开发
  4. 雇佣 Cursor 完成前端
  5. 雇佣测试 AI 验证功能
  6. 自动部署到云服务器
  7. 用户验收确认
```
