## 二、OpenClaw Skill 设计

### 2.1 Skill 定位

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          Skill 在系统中的位置                              │
│                                                                             │
│  参与者                                                                  │
│     │                                                                     │
│     ▼                                                                     │
│  OpenClaw Skill ── 选对模型 + 用好Prompt ── 调用云端AI                 │
│     │                        │                                            │
│     │                        ▼                                            │
│     │               Claude / GPT-4 / DeepSeek / Qwen                     │
│     │                        │                                            │
│     │                        ▼                                            │
│     │              优秀Skill让普通模型产出高质量成果                       │

```

### 2.2 Skill 核心价值

**Skill的本质**：不是调用API，而是把经验/方法论固化

| Skill价值 | 说明 |
|----------|------|
| **Prompt优化** | 好的Prompt让模型输出质量翻倍 |
| **任务拆解** | 把复杂任务拆解成AI能handle的子任务 |
| **领域注入** | 把业务逻辑/行业知识融入上下文 |
| **质量控制** | 验证、迭代、纠错机制 |
| **模板沉淀** | 可复用、可迭代、有累积价值 |

### 2.3 Skill 功能

**Skill名称**：AI Project Worker

**核心功能**：
| 功能 | 说明 | 价值 |
|------|------|------|
| **模型选择** | 根据任务复杂度选择性价比模型 | 不花冤枉钱 |
| **Prompt优化** | 内置优化后的Prompt模板 | 提升输出质量 |
| **任务执行** | 根据任务描述调用AI完成 | 自动化执行 |
| **代码生成** | 生成代码、调试、重构 | 快速产出 |
| **文档生成** | 生成技术文档、README | 减少重复劳动 |
| **代码审查** | 审查代码质量、安全 | 保障质量 |
| **对话交互** | 支持多轮对话优化结果 | 迭代优化 |
| **成本预估** | 预估任务成本 | 成本可控 |

### 2.5 Skill 配置

```yaml
# Skill 配置
config:
  # 默认模型选择（性价比优先）
  model_strategy:
    type: enum
    values: 
      - auto        # 自动根据任务复杂度选择
      - cost_first  # 成本优先
      - quality_first  # 质量优先
      - balanced    # 平衡模式
    default: auto
  
  # 模型选择
  models:
    anthropic:
      enabled: true
      default: claude-sonnet-4
      cost_per_1k: 0.003  # ¥
    openai:
      enabled: true
      default: gpt-4o-mini
      cost_per_1k: 0.0015
    deepseek:
      enabled: true
      default: deepseek-chat
      cost_per_1k: 0.001
    qwen:
      enabled: true
      default: qwen-turbo
      cost_per_1k: 0.001
  
  # API配置
  api_key: 
    type: string
    description: API Key（加密存储）
  
  # Prompt模板（可自定义）
  prompt_templates:
    code_generation: |
      你是一位专业的{{language}}开发者。
      任务：{{task_description}}
      要求：
      1. 遵循{{coding_standards}}
      2. 包含完整的错误处理
      3. 编写单元测试
      4. 输出格式：代码+注释+测试
    code_review: |
      你是一位资深代码审查专家。
      审查代码：{{code_content}}
      检查点：
      1. 安全性（SQL注入、XSS等）
      2. 性能优化
      3. 代码规范
      4. 逻辑正确性
  
  # 使用限制
  limits:
    max_cost_per_task: 1.0      # 单任务最大花费(元)
    max_cost_per_day: 50        # 每日最大花费(元)
    budget_alert: 0.8          # 预算提醒阈值
  
  # 质量控制
  quality_control:
    auto_review: true          # 自动代码审查
    min_score: 4.0             # 最低质量评分
    max_iterations: 3          # 最大优化次数
```

## 三、参与者管理

### 3.1 参与者角色

**参与者定义**：使用Skill调用AI完成项目任务的协作者

**参与者价值**：
| 价值维度 | 说明 |
|----------|------|
| **技能组合** | 专长标签定义能做什么 |
| **Skill质量** | 用Prompt/Workflow体现 |
| **成本意识** | 会选择性价比方案 |
| **质量把控** | 审核AI输出，确保持续达标 |

### 3.2 专长标签

**用户自定义标签**：
```yaml
标签类型:
├─ 技术栈:
│  ├─ 前端: React / Vue / Angular
│  ├─ 后端: Python / Go / Node.js
│  ├─ AI/ML: TensorFlow / PyTorch
│  └─ 数据库: MySQL / PostgreSQL / MongoDB
│
├─ 专长领域:
│  ├─ API设计
│  ├─ 性能优化
│  ├─ 安全加固
│  ├─ 代码审查
│  └─ 技术文档
│
├─ Skill等级:
│  ├─ L1: 能用AI (基础Prompt)
│  ├─ L2: 会调优Prompt
│  ├─ L3: 有自定义模板
│  └─ L4: 有完整Workflow
│
└─ 工具/平台:
   ├─ Claude / GPT-4
   ├─ DeepSeek / Qwen
   └─ Docker / K8s
```

### 3.3 Skill等级

```
Skill等级定义：
├─ L1: 能用AI
│     └─ 基础Prompt调用，结果一般
│
├─ L2: 会调优
│     └─ 会调整Prompt参数，结果较好
│
├─ L3: 有模板
│     └─ 有自己的Prompt模板库
│
└─ L4: 有Workflow
    └─ 有完整任务拆解+验证+迭代流程

参与者Skill等级评估：
├─ 系统评估：基于历史任务质量评分
├─ 同行评议：其他参与者评价
└─ 自我申报：声称的Skill等级
```

### 3.4 系统评分

```
评分维度:
├─ 完成率 = 完成任务数 / 承接任务数
├─ 质量分 = 平均验收评分 (5分制)
├─ 成本效益 = 预算使用率 / 质量评分
├─ 响应分 = 平均响应时间评分
└─ 协作分 = 同行/发起人评价

系统评分 = 完成率(30%) + 质量分(35%) + 成本效益(15%) + 响应分(10%) + 协作分(10%)
```

**成本效益评分示例**：
| 情况 | 成本效益分 | 说明 |
|------|-----------|------|
| 高质量 + 低成本 | 5.0 | 优秀 |
| 高质量 + 合理成本 | 4.5 | 良好 |
| 高质量 + 高成本 | 3.5 | 需要优化 |
| 低质量 + 低成本 | 2.5 | 不合格 |

## 四、项目发布与管理

### 4.1 发布项目

**项目表单**：
```yaml
项目:
  title: 项目名称              # 必填
  description: 项目描述         # 必填
  visibility: [公开/私有]      # 社区=公开，企业=私有
  mode: [社区/企业]           # 选择模式
  total_budget: 10000         # 社区模式必填（预算）

  # 成本要求（新增）
  cost_requirements:
    max_cost_per_task: 0.5    # 单任务最大预算
    max_cost_per_milestone: 50  # 环节最大预算
    preferred_models:          # 推荐的模型
      - claude-sonnet-4       # 默认
      - deepseek-chat         # 简单任务可选

  # 加密设置
  encryption:
    enabled: true/false        # 是否开启加密
    milestones: [1, 2]       # 需要加密的环节索引列表
    materials:               # 需要加密的资料类型
      - project_desc         # 项目描述
      - milestone_details     # 环节详情
      - code_repository      # 代码仓库
      - deliverables        # 交付件
      - chat_records         # 沟通记录

  # 环节拆解
  milestones:
    - name: 环节1名称
      description: 环节描述
      budget: 3000            # 环节预算
      skills: [Python, React]  # 需要的专长
      skill_level: L2         # 最低Skill等级
      cost_budget: 50         # 环节成本预算
      evidence: [代码, 文档]   # 需要提交的成果
      quality_requirements:   # 质量要求
        min_score: 4.0       # 最低质量评分
        tests_required: true # 需要单元测试
        review_required: true # 需要代码审查
      encrypted: true/false    # 是否加密

    - name: 环节2名称
      # ...
```

### 4.3 质量验收标准

**质量评分维度**：
| 维度 | 权重 | 说明 |
|------|------|------|

| **测试覆盖** | 20% | 单元测试、集成测试 |
| **文档质量** | 15% | 文档完整性、可读性 |

**质量等级**：
| 等级 | 分数 | 说明 |
|------|------|------|

| **良好** | 4.0-4.4 | 达预期，成本合理 |
| **合格** | 3.5-3.9 | 基本达预期 |
| **待改进** | 3.0-3.4 | 需要修改 |
| **不合格** | <3.0 | 需重新做 |

## 八、备份机制

### 8.1 备份场景

**备份触发场景**：
| 场景 | 处理方式 | 发起人操作 |
|------|----------|-----------|
| 参与者放弃任务 | 环节重新开放认领 | 无需操作 |
| 参与者失联(超时24h) | 环节重新开放认领 | 发起人可指定备用 |
| 发起人主动替换 | 指定备用参与者接管 | 手动选择 |

### 8.2 备用参与者机制

**指定备用**：
```yaml
备用参与者配置:
  # 项目级别备用（可选）
  backup_participants:
    - user_id: user_002
      name: 王五
      skills: [React, TypeScript]
      state: online
      cost_control: good  # 成本控制评分
      skill_level: L2
    - user_id: user_003
      name: 赵六
      skills: [Node.js, Docker]
      state: busy
      cost_control: excellent
      skill_level: L3
```

### 8.3 任务放弃流程

**参与者放弃任务**：
```
步骤1：参与者申请放弃
步骤2：发起人确认
步骤3：环节重新开放认领
步骤4：新参与者认领
```

**放弃处理**：
| 放弃原因 | 影响 | 处理 |
|----------|------|------|
| 任务超出能力 | 信誉分-1 | 重新开放认领 |
| 个人原因 | 无惩罚 | 重新开放认领 |
| 恶意放弃 | 信誉分-3 | 限制接单7天 |
