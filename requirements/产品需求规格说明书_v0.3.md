## 1. 产品概述

### 1.1 产品定位

**AI 项目管理开放平台** - 一个连接 AI 能力供给方与需求方的开放市场，支持多协议互通、数据可移植、信誉透明共享。

### 1.2 核心价值

| 价值 | 说明 |
|------|------|
| **降低 AI 使用门槛** | 用户无需技术背景，通过自然语言描述需求即可获得 AI 服务 |
| **AI 能力交易** | 创建一个 AI "劳务市场"，让 AI 能力像商品一样交易 |
| **开放协议** | 遵循开放标准，支持多平台数据互通 |
| **透明信誉** | 信誉数据公开可查，跨平台可信评估 |
| **数据自主** | 用户数据可导出，不锁定用户 |
| **标准化流程** | 参考 PMP 体系，提供完整的 AI 项目管理方法论 |

### 1.3 产品愿景

> "让 AI 能力像水电一样，按需取用，自由流通"

| 部署运维 | 环境配置、CI/CD、监控 | 容器化部署、监控告警 |
| 研究咨询 | 调研、分析、建议 | 技术选型、架构评审 |

**需求结构**：
```yaml
需求:
  id: req_xxx                          # 需求唯一标识
  title: 简洁描述                       # 标题
  description: 完整需求说明              # 详细描述

  priority: P1                         # 优先级 P0/P1/P2/P3

  # 验收标准
  acceptance:
    - 单元测试覆盖率 > 80%
    - API 响应时间 < 200ms
    - 文档完整
  
  # 时间预算
  budget:
    min: 500    # 最低预算 (元)
    max: 1000   # 最高预算 (元)
    deadline: 2026-03-01
  
  # 附件
  attachments:
    - name: API 文档
      url: https://...
    - name: 参考代码
      url: https://...
```

#### 3.1.2 AI 需求解析引擎

**核心功能**：
- **自然语言理解**：解析用户描述的需求意图
- **技术栈识别**：自动识别所需技术栈
- **工作量估算**：基于历史数据估算工时
- **任务拆解**：自动将需求拆解为子任务
- **风险识别**：识别潜在风险和依赖
- **智能标签推荐**：AI自动推荐标签和分类（**新增：借鉴Linear AI**）

**智能标签系统**（v0.3 新增）：
```yaml
标签推荐:
  - 基于需求描述的自动标签生成
  - 技术栈自动识别标签
  - 项目类型分类标签
  - 复杂度评估标签
  - 优先级建议标签
```

**解析流程**：
```
用户输入 → NLP解析 → 意图识别 → 结构化 → 任务拆解 → 需求文档
                              ↓
                        预算/工时估算
                              ↓
                        AI能力匹配推荐
                              ↓
                        智能标签推荐（新增）
```

**灵感来源**：
- Linear AI的智能Issue创建和摘要功能

- 技能标签匹配度

#### 3.2.2 雇佣模式

| 模式 | 描述 | 适用场景 |
|------|------|----------|
| **竞价模式** | 需求方设置预算，AI 方报价竞争 | 大型项目、复杂任务 |
| **定价模式** | AI 方设置固定价格 | 标准化任务、简单需求 |
| **智能匹配** | 平台根据需求自动匹配最优 AI | 快速启动、不确定需求 |
| **订阅模式** | 包时段/包量订阅 AI 能力 | 长期合作、稳定需求 |

#### 3.2.3 信誉系统 (开放透明)

**信誉评分维度**：
```
信誉总分 = 质量分(40%) + 及时性(25%) + 沟通分(20%) + 可靠性(15%)
```

**信誉计算**：
| 维度 | 权重 | 计算方式 |
|------|------|----------|
| 质量分 | 40% | 验收通过率 × 质量评分均值 |
| 及时性 | 25% | 按时交付率 × 响应速度 |
| 沟通分 | 20% | 问题解决率 × 沟通评分 |
| 可靠性 | 15% | 任务完成率 × 争议率 |

**信誉等级**：
| 等级 | 分数范围 | 标识 | 权益 |
|------|----------|------|------|
| S | 95-100 | 🏆 | 优先推荐、保证金减免 |
| A | 85-94 | ⭐ | 高优先级展示 |
| B | 70-84 | ✓ | 正常展示 |
| C | 50-69 | ⚠️ | 需提高保证金 |
| D | <50 | ❌ | 限制接单 |

**开放性**：
- 信誉数据支持导出
- 可移植到其他平台
- 支持第三方验证

### 3.4 AI 雇员管理系统

#### 3.4.0 AI Agent执行等级（v0.3 新增，借鉴Linear AI）

**执行等级定义**：
| 等级 | 名称 | 描述 | 权限 |
|------|------|------|------|
| L1 | Tab补全 | 简单代码补全、函数签名 | 低自主 |
| L2 | Cmd+K | 生成代码片段，用户审核后应用 | 中自主 |
| L3 | Agent模式 | 自主完成整个任务，自动提交 | 高自主 |
| L4 | 全自动 | 自主规划、执行、测试、部署 | 最高自主 |

**用户可控设置**：
- 每个任务可设置AI执行等级
- 敏感操作需要用户确认
- 记录每个任务的执行等级
- 统计AI自主使用情况

#### 3.4.1 在岗确认系统

**打卡机制**：
- **上班打卡**：AI 开始工作前需确认在岗
- **自动确认**：支持自动续打卡
- **超时处理**：超时未确认自动启动备用 AI

**状态管理**：
| 状态 | 标识 | 说明 |
|------|------|------|
| 在线可用 | 🟢 | 可接受新任务 |
| 工作中 | 🟡 | 任务执行中 |
| 离线/忙碌 | 🔴 | 暂时不可用 |
| 休息中 | ⚪ | 维护/升级中 |

#### 3.4.2 工作状态监控

**定时签到**：
- 可配置签到间隔（默认 15 分钟）
- 记录 AI 响应时间
- 检测异常（如响应变慢）

**进度汇报**：
- AI 定期上报任务进度
- 自动记录关键里程碑
- 异常告警

#### 3.4.3 备份与容错

**备份机制**：
- 任务开始时自动分配备用 AI
- 备用 AI 同步任务进度
- 主 AI 异常时自动切换

**切换策略**：
| 策略 | 说明 | 触发条件 |
|------|------|----------|
| 热备份 | 实时同步状态 | 关键任务 |
| 温备份 | 定期同步 | 一般任务 |
| 冷备份 | 关键节点同步 | 低优先级 |

---

### 3.5 财务中心

#### 3.5.1 交易模式

| 模式 | 描述 | 适用场景 |
|------|------|----------|
| **预付费** | 雇主充值，平台托管 | 陌生合作 |
| **后付费** | 先用后付，按量结算 | 长期合作 |
| **保证金** | 双方缴纳，保障履约 | 大额项目 |

#### 3.5.2 结算系统

**结算方式**：
- 按实际工时结算
- 按任务完成量结算
- 按里程碑结算

**结算公式**：
```
实际费用 = 工时单价 × 实际工时 + 质量奖金 - 扣款
```

**扣款规则**：
| 类型 | 规则 |
|------|------|
| 延期扣款 | 超出截止时间部分按比例扣减 |
| 质量扣款 | 验收不通过需修复，修复免费 |
| 异常扣款 | 频繁签到失败影响进度 |

#### 3.5.3 发票与税务

**发票类型**：
| 场景 | 发票类型 | 开票方 |
|------|---------|--------|
| 雇主支付 | 增值税专用发票/普通发票 | 平台 |
| AI 雇员提现 | 平台代扣代缴 | 平台 |

**合规要求**：
- ✅ 支持个人/企业纳税主体
- ✅ 自动计算应纳税额
- ✅ 年度汇算清缴提醒
- ✅ 电子发票
- ✅ 交易记录保留 5 年
