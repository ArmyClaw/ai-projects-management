# Agent时代项目团队构建通用范式 v0.9

> 状态：草案可评审
> 适用：基于 Agent 团队协作的软件项目平台
> 核心：先建团队能力，再启动项目执行

---

## 一、目标与边界

### 1.1 目标
- 提供可视化方式创建“项目团队模板”：角色、人数、能力、流程、周期。
- 支持为角色初始化 Agent 员工，并持续扩展其 Skill、MCP 与模型配置能力。
- 通过统一公共能力库（Skill/MCP/Model）实现复用、审计与版本管理。
- 通过项目配置模板一键启动项目，减少重复配置成本。

### 1.2 非目标
- 不负责替代外部代码仓库（GitHub/GitLab）本身功能。
- 不在 v0.9 内实现复杂财务结算与链上支付。
- 不在 v0.9 内实现全自动无人值守研发闭环。

---

## 二、核心对象模型

- Role（角色）: 团队分工定义，如架构师、后端、测试。
- Agent（员工）: 绑定到角色的执行单元，具备可增减的能力集。
- Skill: 任务方法论与提示模板能力，支持版本与标签。
- MCP: 可调用工具能力定义，支持参数、权限和连通性校验。
- Model Profile: 模型配置资产，定义厂商、模型、成本等级、能力标签。
- Team Template: 角色结构 + 人数 + 默认能力 + 计划周期。
- Project Template: 团队模板 + 开发流程 + 里程碑 + 质量门禁。
- Project: 从模板实例化的实际项目。

---

## 三、端到端流程

1. 维护公共 Skill/MCP/Model 库。
2. 创建角色并配置默认 Skill/MCP。
3. 初始化 Agent 员工并补充个人擅长能力与默认模型。
4. 在项目中为同角色多 Agent 指定主负责人与辅助 Agent。
5. 创建项目模板（角色编制、周期、流程、里程碑、门禁）。
6. 选择模板启动项目，自动生成执行计划与团队编组。
6. 执行期持续监控：能力缺口、进度偏差、风险告警。
7. 项目复盘回流：沉淀高价值 Skill/MCP 到公共库。

---

## 四、设计原则

- 单一事实源：Skill/MCP/Model 的定义、版本、状态只在公共库统一维护。
- 模板优先：先标准化模板，再实例化项目。
- 能力与人员解耦：角色定义能力基线，Agent 定义个体增强能力。
- 可观测与可审计：关键动作可追踪（谁在何时改了什么）。
- 渐进自动化：先人机协同，再提高自动化比重。

---

## 五、成熟度模型

- L1 手工编排：手动配置团队和计划，模板复用低。
- L2 模板复用：项目模板可复用，基础流程自动生成。
- L3 能力治理：Skill/MCP 版本治理、审批与兼容性检查。
- L4 智能编排：按目标自动推荐角色配置、周期与能力组合。
- L5 自适应组织：根据项目表现动态调优团队结构与能力。

---

## 六、v0.9 最小落地清单

- 必须有
- 角色管理、Agent 初始化、Skill 库、MCP 库、模型库。
- 团队模板与项目模板。
- 项目启动向导（选择模板 -> 参数确认 -> 启动）。
- 基础看板（进度、角色负载、能力缺口）。

- 可以后置
- 自动排期优化。
- 高级依赖图与成本优化。
- 外部平台深度联动编排。

---

## 七、检查清单

### 启动前
- 角色是否完整覆盖交付流程。
- 每个角色是否至少绑定 1 个 Skill 与 1 个 MCP。
- 每个 Agent 是否绑定默认模型；主负责人是否绑定高能力模型。
- 模板中周期与里程碑是否可执行。

### 执行中
- Agent 负载是否超阈值。
- 关键节点是否按期完成。
- 能力缺口是否已分配补救动作。

### 结束后
- 复盘结论是否回流模板。
- 新 Skill/MCP/Model 是否进入公共库并标注版本。
- 模板是否更新为下一轮可复用版本。

---

## 八、v0.9 一句话总结

这个系统不是“任务看板”，而是“Agent 团队能力编排与项目启动操作系统”。
