## 一、信息架构

- Dashboard（总览）
- Role Center（角色中心）
- Agent Center（员工中心）
- Skill Library（公共 Skill 库）
- MCP Library（公共 MCP 库）
- Model Library（公共模型库）
- Team Templates（团队模板）
- Project Templates（项目模板）
- Project Bootstrap（项目启动向导）
- Project Board（项目看板）
- Audit Logs（审计日志）

---

## 二、核心页面原型

### 2.1 Dashboard
- 展示模板复用率、项目启动成功率、MCP/Model 健康率、Agent 负载热力图。
- 快速入口：新建角色/新建模板/启动项目。

### 2.2 角色中心
- 左侧角色列表，右侧角色详情。
- 角色详情含职责、默认 Skill、默认 MCP、关联 Agent 数量。
- 支持复制角色配置。

### 2.3 员工中心
- 按角色筛选 Agent。
- 每个 Agent 展示：继承能力、个人增强能力、默认模型、当前负载、状态。
- 支持批量分配能力。

### 2.4 Skill 库
- 列表字段：名称、版本、状态、标签、引用次数、更新时间。
- 详情页含定义 JSON 预览与版本历史。
- 支持 `DRAFT -> ACTIVE -> DEPRECATED` 生命周期操作。

### 2.5 MCP 库
- 列表字段：名称、版本、状态、健康状态、最后检查时间。
- 操作：健康检查、发布、下线。
- 详情页展示参数 schema 与权限策略。

### 2.6 Model 库
- 列表字段：名称、厂商、模型ID、tier、状态、健康状态。
- 支持 `DRAFT -> ACTIVE -> DEPRECATED` 生命周期操作。
- 支持健康检查与可用性状态展示。

### 2.7 团队模板页
- 配置角色编制（角色 + 人数）。
- 展示每角色能力包与默认模型策略摘要。
- 支持模板复制与发布。

### 2.8 项目模板页
- 关联团队模板。
- 配置周期、阶段、里程碑、质量门禁。
- 保存草稿与发布版本。

### 2.9 项目启动向导

步骤 1：选择项目模板
步骤 2：填写项目参数（名称、起始日期、目标）
步骤 3：按角色配置 Agent 实例与主辅关系（1主多辅）
步骤 4：校验依赖（角色、Skill、MCP、Model）
步骤 5：确认并启动

- 若校验失败：显示可修复项列表。
- 若校验成功：跳转项目看板。

### 2.10 项目看板
- 左侧：阶段与里程碑。
- 中部：任务泳道（待办/进行中/完成）。
- 右侧：团队负载与能力缺口告警。

### 2.11 审计日志
- 支持按实体类型筛选（Role/Agent/Skill/MCP/Model/Template/Project）。
- 显示操作人、变更字段、时间、上下文。

---

## 三、交互规则

- 关键变更操作要求二次确认（发布、下线、废弃）。
- 校验失败必须给出“定位 + 修复建议”。
- 状态颜色统一：
- `ACTIVE` 绿色，`DRAFT` 灰色，`DEPRECATED` 橙色，`BLOCKED` 红色。

---

## 四、响应式建议

- Desktop 优先（>= 1280px）。
- Tablet 支持模板与库管理。
- Mobile 仅提供查看与轻量审批，不支持复杂编辑。

---

## 五、前端实现建议

- Vue 3 + TypeScript + Pinia + Vue Router。
- 组件分层：
- 领域组件（RoleCard/SkillSelector/McpHealthBadge）。
- 领域组件（RoleCard/SkillSelector/McpHealthBadge/ModelTierTag/RoleAgentAssignmentEditor）。
- 页面容器（RolePage/BootstrapWizard）。
- 基础组件（表格、抽屉、确认框）。
