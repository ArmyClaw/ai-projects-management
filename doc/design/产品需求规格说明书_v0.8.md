# Agent Team Builder - 产品需求规格说明书 v0.9

> 文档版本：v0.9
> 更新日期：2026-03-01
> 产品定位：Agent 项目团队创建与项目启动 Web 可视化系统

---

## 一、产品定位

### 1.1 愿景
把“组建项目团队 + 配置能力 + 启动执行”标准化为可复用模板，让项目从想法到启动可在 10 分钟内完成。

### 1.2 目标用户
- 平台管理员：维护公共 Skill/MCP/模型库与系统配置。
- 组织负责人（项目发起人）：定义角色结构、模板并启动项目。
- 团队管理者：维护 Agent 员工与能力分布。

### 1.3 核心价值
- 从“人找任务”升级为“能力驱动的团队编排”。
- 降低重复配置，提升项目启动效率。
- 让团队能力沉淀为可治理的资产。

---

## 二、核心流程

1. 管理员维护公共 Skill/MCP/模型库。
2. 发起人创建角色并配置默认能力。
3. 管理者初始化 Agent 员工，追加擅长能力并绑定默认模型。
4. 在项目角色下配置多 Agent 实例，指定主负责人与辅助 Agent。
5. 创建项目配置模板（角色编制、周期、流程、门禁）。
6. 从模板启动项目，生成初始计划与执行看板。

---

## 三、功能需求

### 3.1 角色管理
- 创建/编辑/归档角色。
- 为角色绑定默认 Skill 列表与 MCP 列表。
- 定义角色基础职责与交付物。

### 3.2 Agent 员工管理
- 初始化 Agent（名称、角色、能力基线）。
- 为 Agent 增加/移除擅长 Skill 与 MCP。
- 为 Agent 绑定默认模型配置（来自公共模型库）。
- 显示 Agent 负载、可用状态、历史参与项目。

### 3.3 公共模型库
- 模型配置管理（厂商、模型ID、上下文长度、成本等级、能力标签）。
- 模型分级（`PREMIUM`/`BALANCED`/`ECONOMY`）与默认用途。
- 模型可用性校验与状态管理（可用/降级/不可用）。
- 模型引用关系（被哪些 Agent/模板/项目使用）。

### 3.4 公共 Skill 库
- Skill 的新增、版本化、标签化、启停管理。
- Skill 兼容性标记（适用角色/适用场景）。
- Skill 引用关系（被哪些角色/模板使用）。

### 3.5 公共 MCP 库
- MCP 定义管理（名称、描述、参数 schema、权限范围）。
- 连通性验证（可用/不可用/降级）。
- 安全策略（可见范围、调用白名单）。

### 3.6 团队模板
- 定义角色编制（角色 + 人数）。
- 定义每角色的默认能力包（Skill + MCP）。
- 定义角色默认模型策略（主负责人模型档位、辅助模型档位）。
- 支持模板版本与复制。

### 3.7 项目模板
- 关联团队模板。
- 定义开发周期、阶段流程、里程碑。
- 定义质量门禁（评审点、通过条件）。

### 3.8 项目启动
- 启动向导：选择模板 -> 填项目参数 -> 生成项目。
- 自动初始化团队与计划。
- 对每个角色可配置多个 Agent 实例并设置 1 个主负责人。
- 输出项目看板与后续执行入口。

### 3.9 监控与治理
- 进度偏差、角色负载、能力缺口监控。
- 变更审计：模板/角色/能力调整历史。

## 四、关键业务规则

### 4.1 能力约束
- 每个角色至少绑定 1 个 Skill。
- 涉及外部工具调用的角色至少绑定 1 个 MCP。
- 下线中的 Skill/MCP 不允许新模板引用。
- 每个 Agent 必须绑定 1 个默认模型。

### 4.2 主辅协作约束
- 同一项目角色可配置多个 Agent 实例。
- 每个角色必须且仅可指定 1 个 `PRIMARY` Agent。
- `PRIMARY` Agent 必须绑定 `PREMIUM` 或 `BALANCED` 模型档位。
- `ASSISTANT` Agent 可绑定 `BALANCED` 或 `ECONOMY` 模型档位。

### 4.3 模板约束
- 团队模板必须包含至少 1 个角色。
- 项目模板必须定义周期与至少 1 个里程碑。
- 删除模板采用软删除，不影响已启动项目。

### 4.4 项目启动约束
- 启动时校验能力依赖完整性。
- 若关键 MCP 不可用，项目状态为 `BLOCKED`。
- 若主负责人模型不可用，项目状态为 `BLOCKED`。
- 项目创建后自动生成初始执行计划。

## 五、状态机

### 5.1 Skill/MCP 状态
- `DRAFT` -> `ACTIVE` -> `DEPRECATED` -> `ARCHIVED`

### 5.2 Model 状态
- `DRAFT` -> `ACTIVE` -> `DEPRECATED` -> `ARCHIVED`

### 5.3 模板状态
- `DRAFT` -> `PUBLISHED` -> `DEPRECATED`

### 5.4 项目状态
- `INIT` -> `PLANNED` -> `ACTIVE` -> `ON_HOLD` -> `COMPLETED` / `CANCELLED`

## 六、MVP 范围（v0.9）

### 6.1 必做
- 角色管理
- Agent 初始化与能力维护
- 公共模型/Skill/MCP 库
- 团队模板与项目模板
- 项目启动向导
- 基础看板与审计日志

### 6.2 延后
- 自动排期优化
- 智能能力推荐
- 跨组织模板市场

---

## 七、验收标准

- 10 分钟内可完成“从模板选择到项目启动”。
- 至少支持 3 类角色模板（研发、测试、管理）。
- 能够追踪任意 Skill/MCP/模型 被谁何时修改。
- 同角色 3 个 Agent 场景下可正确设置 1 主 2 辅并通过启动校验。
- 项目启动失败时给出可定位的缺失项清单。

---

## 八、风险与应对

- 风险：能力库膨胀导致治理失控。
- 应对：版本冻结、标签规范、过期清理规则。

- 风险：MCP 稳定性影响启动成功率。
- 应对：启动前连通性检查 + 降级策略。

- 风险：模板和真实执行脱节。
- 应对：项目复盘强制回写模板差异。

---

## 九、版本更新点（相对 v0.8）

- 聚焦 Agent 团队创建与项目启动，不再以交易结算为主线。
- 明确 Skill 与 MCP 均为一等能力对象，统一治理。
- 新增公共模型库与 Agent 级模型配置。
- 新增“同角色多 Agent 主辅协作”规则。
- 建立团队模板 + 项目模板双层复用机制。
- 新增项目启动状态机与关键校验规则。
