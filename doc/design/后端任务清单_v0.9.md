# Agent Team Builder - 后端任务清单 v0.9

> 目标：按迭代落地后端能力（含模型库与主辅 Agent 分配）

---

## Sprint 1：数据层与基础能力

1. 建表与迁移
- 新增 `models`、`project_role_agents`。
- `agents` 增加 `default_model_id`。
- 增加唯一部分索引：每个 `project_id + role_id` 仅1个 `PRIMARY`。

2. 枚举与约束
- 新增 `model_tier`、`project_agent_role`。
- 约束：`PRIMARY` 模型 tier 只能 `PREMIUM/BALANCED`。

3. 审计扩展
- `entity_type` 增加 `MODEL`。
- 落库模型发布、健康检查、项目角色分配变更。

验收：
- 迁移可重复执行。
- 约束在 DB 层可拦截非法数据。

---

## Sprint 2：Model API 与 Agent 模型绑定

1. Model API
- `POST /models`
- `GET /models`
- `GET /models/:id`
- `POST /models/:id/health-check`
- `POST /models/:id/publish`

2. Agent 模型绑定
- `PATCH /agents/:id/model`
- 创建 Agent 时支持 `defaultModelId`。

3. 校验逻辑
- 模型必须 `ACTIVE` 才可绑定。
- 不可用模型返回 `MODEL_UNAVAILABLE`。

验收：
- Model 生命周期完整跑通。
- Agent 模型更新可审计。

---

## Sprint 3：项目主辅分配与启动校验

1. 分配接口
- `PUT /projects/:id/role-agents`

2. 启动校验增强
- `POST /projects/bootstrap/validate`
- 校验每角色 `PRIMARY` 唯一。
- 校验 `PRIMARY` 模型档位。
- 校验模型可用性。

3. 启动落库
- `POST /projects/bootstrap`
- 落库 `project_role_agents`。

验收：
- 同角色 3 Agent 场景可通过（1主2辅）。
- 缺失主负责人时返回 `MISSING_PRIMARY_AGENT`。

---

## Sprint 4：稳定性与质量

1. 单元测试
- 模型状态机。
- 主辅约束服务层。
- 启动校验错误分支。

2. 集成测试
- create model -> publish -> bind agent -> bootstrap。

3. 观测指标
- 模型可用率。
- 启动失败原因分布。

验收：
- 核心接口覆盖率 >= 80%。
- 启动链路 P95 满足目标。
