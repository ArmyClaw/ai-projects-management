## 二、路由分层

### 2.1 公共路由
- `login`

### 2.2 受保护路由（需登录）
- `app/*`

### 2.3 权限路由
- `ORG_ADMIN`：全量。
- `TEAM_MANAGER`：`roles/agents/team-templates`。
- `PROJECT_OWNER`：`project-templates/projects/bootstrap/projects/*`。
- `VIEWER`：`dashboard/audit-logs(只读)/projects(只读)`。

## 三、页面级实现清单

## 3.1 Dashboard

核心模块：
- 启动成功率卡片
- MCP 健康率卡片
- 模板复用率卡片
- Agent 负载热力图

依赖接口：
- `GET /dashboard/metrics`（建议新增聚合接口）

---

## 3.2 角色中心 `/roles`

核心模块：

---

## 3.8 项目启动向导 `/projects/bootstrap`

步骤页：
1. 选择模板
2. 填写项目参数
3. 配置角色 Agent 实例（主负责人 + 助手）
4. 校验依赖
5. 启动确认

依赖接口：
- `GET /project-templates?status=PUBLISHED`
- `POST /projects/bootstrap/validate`
- `POST /projects/bootstrap`

关键交互：
- 校验失败显示“字段定位 + 修复建议”。
- 启动成功后跳转 `/app/projects/:id/board`。
- 同角色仅允许设置 1 个 `PRIMARY`。

## 六、前端任务拆分建议（两周）

### 第1周
- D1-D2：路由骨架 + 权限守卫 + 布局框架。
- D3-D4：Role/Agent 页面闭环（含默认模型绑定）。
- D5：Skill/MCP/Model 列表与详情。

### 第2周
- D6-D7：团队模板/项目模板页面。
- D8-D9：项目启动向导（含主辅分配与失败分支）。
- D10：项目看板和审计日志基础版。
