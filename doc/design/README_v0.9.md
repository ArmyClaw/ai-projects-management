## 文档职责
- 通用范式：方法论与边界。
- PRD：功能范围、规则、MVP 与验收。
- 架构：数据模型、API、启动编排、权限与审计。
- 场景：端到端业务流与成功标准。
- 原型：页面结构、交互与实现建议。
- 数据字典：字段、约束、索引与归档策略。
- API契约：请求响应、错误码、幂等与限流。
- 前端清单：页面树、路由、组件、状态与迭代计划。
- 后端任务清单：后端按 Sprint 的可执行落地任务。
- 前端任务清单：前端按 Sprint 的可执行落地任务。
- 联调用例：前后端联调主流程与失败分支验收。
- Notion风原型：黑白极简视觉规范与关键页面线框稿。
