# Agent Team Builder - 架构设计文档 v0.9

> 文档版本：v0.9
> 更新日期：2026-03-01
> 目标：支持角色/Agent/Skill/MCP/Model/模板/项目启动全链路

---

## 一、架构总览

### 1.1 分层
- 前端：Web 可视化管理台（角色、能力库、模板、项目启动）。
- 后端：REST API + 编排服务（校验与初始化）。
- 数据层：PostgreSQL（主数据）+ Redis（短期缓存/任务队列可选）。
- 集成层：MCP 连通性探测与调用网关。
- 模型层：统一模型配置中心（可用性探测、分级与路由策略）。

### 1.2 核心服务
- IAM 服务：用户、组织、角色权限。
- Capability 服务：Skill/MCP/Model 生命周期管理。
- Template 服务：团队模板与项目模板。
- Project Bootstrap 服务：项目启动校验与初始化。
- Audit 服务：变更审计与查询。

## 二、核心数据模型

### 2.1 主要实体
- `organizations`
- `users`
- `roles`
- `agents`
- `skills`
- `mcps`
- `models`
- `team_templates`
- `project_templates`
- `projects`
- `project_plans`
- `project_role_agents`
- `audit_logs`

### 2.2 关键关系
- 角色与 Skill/MCP：多对多。
- Agent 与 Skill/MCP：多对多（继承角色基线 + 个体增强）。
- Agent 与 Model：多对一（默认模型）+ 项目内覆盖配置。
- 团队模板与角色编制：一对多。
- 项目模板与团队模板：多对一。
- 项目与模板：多对一（实例化关系）。
- 项目角色与 Agent：一对多，且每角色仅 1 个主负责人。

### 2.3 表结构示例（Prisma）

```prisma
model Role {
  id            String   @id @default(uuid())
  orgId         String
  name          String
  description   String?
  status        RoleStatus @default(ACTIVE)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  roleSkills    RoleSkill[]
  roleMcps      RoleMcp[]
  agents        Agent[]
}

model Agent {
  id            String   @id @default(uuid())
  orgId         String
  roleId        String
  name          String
  workload      Int      @default(0)
  status        AgentStatus @default(ACTIVE)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  role          Role     @relation(fields: [roleId], references: [id])
  agentSkills   AgentSkill[]
  agentMcps     AgentMcp[]
  defaultModelId String?
}

model Skill {
  id            String   @id @default(uuid())
  orgId         String
  name          String
  version       String
  status        CapabilityStatus @default(DRAFT)
  tags          String[]
  definition    Json
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}

model Mcp {
  id            String   @id @default(uuid())
  orgId         String
  name          String
  version       String
  status        CapabilityStatus @default(DRAFT)
  endpoint      String
  paramSchema   Json
  authPolicy    Json
  healthStatus  HealthStatus @default(UNKNOWN)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}

model ModelConfig {
  id            String   @id @default(uuid())
  orgId         String
  name          String
  provider      String
  modelId       String
  tier          ModelTier // PREMIUM/BALANCED/ECONOMY
  status        CapabilityStatus @default(DRAFT)
  maxContext    Int?
  pricing       Json?
  healthStatus  HealthStatus @default(UNKNOWN)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}
```

## 四、启动编排逻辑

### 4.1 校验步骤
1. 校验模板状态必须为 `PUBLISHED`。
2. 校验角色编制完整。
3. 校验引用 Skill/MCP 均为 `ACTIVE`。
4. 对关键 MCP 执行连通性检测。
5. 校验每个角色的主负责人唯一且模型等级符合策略。
6. 校验主负责人模型可用性（`HEALTHY/DEGRADED`）。
7. 生成项目计划初稿与里程碑。

### 4.2 失败返回规范

```json
{
  "ok": false,
  "code": "BOOTSTRAP_VALIDATION_FAILED",
  "errors": [
    { "field": "mcps[2]", "reason": "HEALTH_CHECK_FAILED" },
    { "field": "roles[qa]", "reason": "MISSING_SKILL" },
    { "field": "roles[backend]", "reason": "MISSING_PRIMARY_AGENT" },
    { "field": "agents[qa-01].model", "reason": "MODEL_UNAVAILABLE" }
  ]
}
```

## 五、权限模型

- `ORG_ADMIN`：全量管理能力库、模板、项目。
- `TEAM_MANAGER`：管理角色、Agent、团队模板。
- `PROJECT_OWNER`：管理项目模板并启动项目。
- `VIEWER`：只读。

权限原则：最小权限 + 操作留痕。

## 七、非功能要求

- 可用性：核心 API 月可用性 >= 99.9%。
- 性能：启动校验接口 P95 < 1.5s（不含外部网络抖动）。
- 安全：MCP 凭据不明文落库，采用密文存储。
- 可维护：所有状态变更必须产生日志。
