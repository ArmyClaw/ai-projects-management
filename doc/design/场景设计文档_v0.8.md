## 场景一：管理员维护公共 Skill 库

### 场景描述
管理员新增一个“后端 API 设计”Skill，发布到公共库，供角色和模板复用。

### 关键流程
1. 进入 Skill 库，点击“新建 Skill”。
2. 填写名称、版本、标签、定义。
3. 保存为 `DRAFT` 并进行预览验证。
4. 审核通过后发布为 `ACTIVE`。
5. 系统记录审计日志。

### 成功标准
- Skill 可被角色配置页检索并绑定。
- 审计日志可追溯发布人和时间。

## 场景二：管理员维护公共 MCP 库

### 场景描述
管理员新增 MCP 工具并进行连通性检测。

### 关键流程
1. 新建 MCP，配置 endpoint、参数 schema、鉴权策略。
2. 保存为 `DRAFT`。
3. 点击健康检查，状态变为 `HEALTHY` 或 `UNHEALTHY`。
4. 仅 `HEALTHY` 且审核通过的 MCP 可发布 `ACTIVE`。

### 成功标准
- 发布前强制通过健康检查。
- 项目启动仅允许引用 `ACTIVE` MCP。

## 场景三：团队管理者创建角色并配置能力基线

### 场景描述
团队管理者创建“测试工程师”角色并绑定默认 Skill/MCP。

### 关键流程
1. 创建角色并填写职责说明。
2. 绑定默认 Skill（如测试策略、用例设计）。
3. 绑定默认 MCP（如测试执行工具）。
4. 保存角色。

### 成功标准
- 角色可被团队模板引用。
- 角色能力满足最小约束（至少 1 Skill）。

## 场景四：初始化 Agent 员工并追加擅长能力

### 场景描述
为“测试工程师”角色初始化 2 个 Agent，并给其中 1 人增加性能测试专长与更高模型档位。

### 关键流程
1. 在角色下创建 Agent 员工。
2. 系统继承角色默认能力。
3. 手动追加个人擅长 Skill/MCP。
4. 为 Agent 绑定默认模型（公共模型库中选择）。
4. 查看负载与可用状态。

### 成功标准
- Agent 能力视图区分“角色继承”和“个人增强”。
- 变更可审计。

## 场景四A：同角色多 Agent 主辅协作配置

### 场景描述
后端角色配置 3 个 Agent：1 个主负责人使用高能力模型，2 个助手使用高性价比模型。

### 关键流程
1. 在项目角色配置中选择 3 个后端 Agent。
2. 指定 `backend-lead` 为 `PRIMARY`，绑定 `PREMIUM` 模型。
3. 指定 `backend-assistant-1/2` 为 `ASSISTANT`，绑定 `ECONOMY` 模型。
4. 系统执行规则校验（唯一主负责人、模型可用性、档位约束）。

### 成功标准
- 同角色只存在 1 个主负责人。
- 主负责人模型可用且档位符合策略。
- 助手模型可用且成本更优。

## 场景五：创建团队模板

### 场景描述
项目负责人创建“标准 Web 研发团队模板”。

### 关键流程
1. 添加角色编制：架构师 1、后端 2、前端 2、测试 1。
2. 确认各角色能力包。
3. 保存模板为 `DRAFT`。
4. 发布模板为 `PUBLISHED`。

### 成功标准
- 模板可被项目模板引用。
- 模板支持版本化和复制。

## 场景六：创建项目模板

### 场景描述
在团队模板基础上定义“6 周迭代开发项目模板”。

### 关键流程
1. 选择团队模板。
2. 配置周期（6 周）和阶段（需求/开发/测试/上线）。
3. 设置里程碑与质量门禁。
4. 发布项目模板。

### 成功标准
- 模板包含周期、里程碑、门禁。
- 可直接用于项目启动。

## 场景七：按模板启动项目

### 场景描述
发起人通过启动向导创建真实项目。

### 关键流程
1. 选择项目模板。
2. 输入项目名称、开始日期、目标说明。
3. 系统执行依赖校验（角色/Skill/MCP/Model 状态与连通性）。
4. 校验通过后创建项目与初始计划。
5. 跳转项目看板。

### 失败分支
- MCP 健康检查失败 -> 项目置为 `BLOCKED`，提示替换或重试。
- 角色能力缺失 -> 返回缺失清单并阻止启动。
- 角色未指定主负责人 -> 返回 `MISSING_PRIMARY_AGENT`。
- 主负责人模型不可用 -> 返回 `MODEL_UNAVAILABLE`。

### 成功标准
- 可生成可执行的初始计划。
- 启动失败信息可定位可修复。

## 场景八：项目复盘回流模板

### 场景描述
项目结束后将有效实践回流为模板更新。

### 关键流程
1. 复盘页面对比“模板计划 vs 实际执行”。
2. 标记新增高价值 Skill/MCP。
3. 提交模板优化建议。
4. 审批后生成模板新版本。

### 成功标准
- 模板持续进化。
- 公共能力库得到真实项目反馈。

---

## 统一验收口径

- 任何“发布”动作必须可审计。
- 任何“启动”动作必须有完整校验日志。
- 任何“模板”都必须可复制、可废弃、可追踪版本。
